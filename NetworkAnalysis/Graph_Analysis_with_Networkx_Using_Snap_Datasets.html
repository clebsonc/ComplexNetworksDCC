Graph Analysis with Networkx Using Snap Datasets¶

by: Clebson C. Alves de Sá

In this Notebook I will assess three large networks that can be freely downloaded from the Stanford Network Analysis project -- Snap. The three datasets are represented as undirected graphs, and a small description of them follows bellow:

  • Social circles from Facebook: This dataset consists of 'circles' or 'friend lists' from Facebook. The complete description with statistics of the dataset can be download here.
  • General Relativity and Quantum Cosmology Collaboration: This dataset consists of the collaboration network from the e-print arxiv. The network is composed of scientific collaborations between authors. Nodes are authors, and a edge of node $i$ to node $j$ implies a collaboration of these two authors. The complete description with the summary of statistics of the dataset can be seen here.
  • High Energy Physics - Theory Collaboration network: Just like the previous network, the third dataset consists of the collaboration networks from arxiv, and covers collaborations between authors that submited papers to the High Energy Physics. The complete description and summary of statistics can be seem here.

Node Degree¶

The node degree corresponds to the amount of neighbors that is linked to a node. It varies according to the two following properties:

  • Undirected Graph: Is the amount of edges that connect a node $k_{i}$ to its neighbors.
  • Directed Graph: Is the amount of incoming and outgoing edges of a Node. Thus, the degree of a node $k_i$ is $k_{i}^{out}+k_{i}^{in}$.

To retrieve the degree of each node is quite simple using networkx, the only thing necessary is to call the builtin function networkx.degree() in the library. The function receives as argument a graph and as a result returns a dictionary containing the degree of each node. Given the dictionary, we transform the data to a dataframe. We show the degree for the 5 first nodes bellow.

Clustering Coefficients¶

The clustering coefficient captures the degree to which the neighbors of a given node link to each other. In other words, for unweighted graphs, the degree of a node $\textit{u}$ is the fraction of possible triangles through that node that exist. In networkx, for undirected graphs, the following equations is used to compute the clustering coefficients:

$c_{u} = \frac{2T(u)}{deg(u)(deg(u) - 1)}$,
where $T(u)$ is the number of triangles through node u and $deg(u)$ is the degree of $u$.

Just like the previous metric, the degree distribution is easilly computed by calling the builtin function nx.clustering(). The clustering coefficient for each node indicates the local density of each node. This implies that if a node is connected/neighboor of every other node in the network, the higher will be its clustering coefficient.

Since the clustering coefficients tells the probability that two neighbors of a node link to each other, we can infer some hypothesis about the three graphs. Firstly, for the facebook social network, it looks like that the clustering coefficient assume a gaussian distribution. This coefficient tell us that the friendship relationship is not strong, and it forms groups of friends. In other words, we can infer that if a node $u$ is friend of a node $v$, and this node $v$ is friend of a node $w$, than it's not the case that the node $u$ is friend of node $w$ as well.

Second, for both collaboration networks, we can visualize that the frequency is higher for higher coefficients, implying more interconnected groups. This indicates that connections are much more stronger, and forms communities of collaborations. Althought there is a higher percentage of higher clustering coefficients in both collaboration networks, we can't ignore the fact that there is as well a large value of nodes with small clustering coefficients. This can be explained by the fact that authors have distinct groups of research.

Betweenness Centrality¶

The betweenness centrality is a measure of centrality in a graph based on shortest paths over all nodes in the graph, therefore, it gives the degree of importance of which nodes stand between each other. In other words, the betweenness centrality defines the centrality of a node. In networkx, there is the builtin function nx.betweenness_centrality() that returns the centrality for each node.

In [10]:

Closeness Centrality¶

The closeness gives the distance to all other nodes in the graph, or in the case that the graphs is not connected to all other nodes, the value is based on the component that has the node. Just like the betweenness centrality, the closeness is a measure of centrality in a network, which is computed as the sum of the length of the shortest paths between the node and all other nodes in the graph.

Diameter¶

The diameter of a graph is the maximum eccentricity of a graph. The eccentricity is the maximum distance from a note to all remaining nodes. Since there are many components for graph 2 and graph 3, the diameter is the maximum eccentricity of the subgraphs.

In networkx, the diameter is computed by calling the builtin function nx.diameter(). The code bellow shows the diameter for all three graphs:

In [22]:
diameter_graph1 = list()
for subgraph in con_comp_graph1:
    diameter_graph1.append(nx.diameter(subgraph))
diameter_graph2 = list()
for subgraph in con_comp_graph2:
    diameter_graph2.append(nx.diameter(subgraph))
diameter_graph3 = list()
for subgraph in con_comp_graph3:
    diameter_graph3.append(nx.diameter(subgraph))
print("Maximum diameter for Graph1: ", max(diameter_graph1))
print("Maximum diameter for Graph2: ", max(diameter_graph2))
print("Maximum diameter for Graph3: ", max(diameter_graph3))

Maximum diameter for Graph2:  17
Maximum diameter for Graph3:  18

As we can observe, the graph with maximum diameter is the third graph with a diameter of 18 followed by the second graph with diameter of 17. The graph representing facebook has the smaller diameter of 8.
